# Guia de Contribuição

Para manter a qualidade do código da Secretaria de Educação:

1. **Branching:** Use `feature/nome-da-funcionalidade` ou `bugfix/correcao-erro`.
2. **Commits:** Devem ser semânticos (ex: `feat: adiciona alerta de faltas`).
3. **Testes:** Todo PR deve manter o mínimo de 70% de cobertura de testes.
4. **Padrão de Código:** - Mobile: Seguir as guidelines oficiais (Google/Apple).
   - Backend: Seguir SOLID e Clean Architecture.
